What is CI/CD?

CI/CD means Continuous Integration and Continuous Delivery or Deployment. CI automatically tests code changes after every push. This helps developers detect errors early and keep the project stable. CD then publishes the successful build automatically. This process makes cloud applications faster and more reliable.

My Pipeline

When I push to the main branch, GitHub Actions runs automatically. It checks that required files exist and prepares the website files. After that, the workflow deploys the website to GitHub Pages. Finally, the live site updates automatically.
